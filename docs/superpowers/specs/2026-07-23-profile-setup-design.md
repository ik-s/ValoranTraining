# 프로필 설정 UX 설계

## 목표

Google 로그인 사용자가 처음 로그인했을 때 닉네임과 프로필 사진을 설정하게 하고,
이후에는 상단바를 복잡하게 만들지 않으면서 자신의 계정을 관리할 수 있게 한다.

## 사용자 흐름

1. Google 로그인 후 프로필 완료 상태가 아니면 `프로필 설정` 화면으로 이동한다.
2. 화면에서 닉네임을 입력하고, Google 계정 사진을 기본값으로 확인하거나 새 사진을 업로드한다.
3. 저장하면 홈으로 이동한다.
4. 로그인한 뒤 상단바의 `SETTINGS` 오른쪽에는 원형 프로필 사진만 보인다.
5. 사진을 클릭하면 작은 계정 메뉴가 열리며 닉네임, `프로필 설정`, `LOG OUT`을 제공한다.

## 화면 원칙

- 기존의 어두운 패널, 얇은 테두리, 청록색 보조색과 붉은색 주요 버튼을 그대로 사용한다.
- 상단바에서는 텍스트형 계정 컨트롤과 별도 `PROFILE` 버튼을 제거한다.
- 프로필 사진은 실제 Google 사진 또는 사용자가 올린 파일만 표시한다. 사진이 없으면 접근 가능한 기본 이미지 자리를 사용한다.
- 메뉴는 사진 버튼에만 연결되고, 바깥을 클릭하거나 Escape를 누르면 닫힌다.
- 모바일에서는 사진 버튼을 유지하고, 계정 메뉴가 화면 밖으로 잘리지 않도록 오른쪽에 정렬한다.

## 데이터와 보안

- `profiles`에 `avatar_url`, `profile_completed`, `updated_at`을 추가한다.
- 기존 로그인 사용자는 `profile_completed = false`로 처리하여 다음 로그인 시 설정 화면을 거친다.
- `profile-avatars` Storage 버킷에 사용자 ID별 경로로 이미지를 저장한다.
- 이미지 형식은 JPEG, PNG, WebP만 허용하고, 클라이언트에서 2MB 이하만 전송한다.
- Storage 정책은 로그인 사용자가 자신의 `<user-id>/` 경로에만 업로드·교체·삭제할 수 있게 제한한다. 읽기는 프로필 사진 표시를 위해 공개 URL로 제공한다.
- 프로필 업데이트 정책은 본인 행의 `display_name`, `avatar_url`, `profile_completed`, `updated_at`만 변경할 수 있도록 제한한다.

## 완료 기준

- 첫 Google 로그인은 프로필 설정으로 이동한다.
- 닉네임과 사진을 저장하면 헤더 사진과 이름이 즉시 반영된다.
- 사진 버튼 클릭 메뉴에서 프로필 설정과 로그아웃이 동작한다.
- 비로그인 상태는 기존 Google 로그인 버튼을 유지한다.
- 단위 테스트, TypeScript 빌드, Supabase 보안 점검을 통과한다.
