# 사용자 안내 README 개편 설계

## 목표

`Valoran Training`을 처음 접하는 일반 사용자가 공개 사이트에 바로 접속하고, 감도 설정부터 훈련 시작까지의 흐름을 이해할 수 있는 한국어 README를 만든다. 개발 환경 구성, 테스트, 빌드 명령은 README에서 제외한다.

## 대상 독자와 범위

- 대상: 데스크톱에서 VALORANT 에임 훈련을 하려는 일반 사용자
- 언어: 한국어 중심, 모드 고유명사는 영어와 한글 설명을 함께 표기
- 포함: 공개 사이트 주소, 실제 서비스 화면, 시작 방법, 훈련 모드, 감도 안내, 조작법, 지원 환경, 비공식 프로젝트 안내
- 제외: 개발 명령, 프로젝트 구조, 테스트 방법, 배포 과정, 내부 저장소 키

## README 구조

1. 서비스 이름과 한 줄 소개
2. 공개 사이트로 이동하는 눈에 띄는 링크
3. 실제 운영 화면 캡처 두 장
   - 홈 또는 훈련 선택 화면
   - 훈련이 실행 중인 3D 화면
4. `시작하기` 3단계
   - 실제 마우스 DPI와 VALORANT 인게임 감도 입력
   - 크로스헤어 설정
   - 모드, 난이도, 30초 또는 60초 훈련 선택
5. 6개 훈련 모드의 짧은 설명
6. 감도와 DPI 안내
   - DPI 입력값은 하드웨어 DPI를 변경하지 않음
   - eDPI와 360° 거리 계산에 사용됨
   - 실제 마우스 소프트웨어 DPI와 VALORANT 감도를 입력해야 함
7. 조작법 및 지원 환경
8. Riot Games 및 VALORANT와 무관한 비공식 팬 프로젝트라는 고지

## 화면 자산

- 공개 배포 사이트에서 실제 동작 중인 화면을 캡처한다.
- 캡처 이미지는 저장소의 문서 자산 폴더에 PNG로 보관하고 README에서 상대 경로로 참조한다.
- 디자인 목업이나 임시 이미지를 사용하지 않는다.

## .gitignore 정책

다음은 로컬 환경·생성물·비밀값이므로 무시한다.

- 의존성 및 생성물: `node_modules/`, `dist/`, `coverage/`, `.vite/`
- 로컬 환경 변수: `.env`, `.env.*` (단, `.env.example`은 추적)
- 로그와 운영체제 파일: `*.log`, `.DS_Store`
- 로컬 분석 캐시: `.code-review-graph/`

다음은 저장소에서 관리한다.

- `.openai/hosting.json`: 배포 프로젝트 식별자만 보관하며 토큰이나 비밀값을 넣지 않음
- `docs/`, `scripts/`, `tests/`: 사용자 문서, 반복 작업, 회귀 검증에 필요

## 검증 기준

- README에 개발자용 실행·테스트 명령이 없다.
- 공개 사이트 링크와 실제 화면 캡처가 모두 표시된다.
- 감도 안내가 앱의 실제 동작과 일치한다.
- `.gitignore`는 필요한 소스·문서·테스트·배포 식별자를 제외하지 않는다.
