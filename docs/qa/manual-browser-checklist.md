# 수동 브라우저 QA 체크리스트

대상: Windows 데스크톱 Chrome, 실제 마우스, WebGL 사용 가능 환경. 각 항목의 결과와 브라우저 버전을 릴리스 노트에 기록한다.

## 시작 전

- [ ] `npm run build`가 성공한다.
- [ ] `npm run dev`로 로컬 앱을 열어 첫 화면, 감도 입력, eDPI 실시간 계산(800 × 0.32 = 256)을 확인한다.
- [ ] 모바일 폭/터치 환경에서는 데스크톱 요구 안내가 표시된다.
- [ ] 설정과 크로스헤어를 새로고침 뒤에도 유지한다.

## 입력과 세션 수명

- [ ] 훈련 시작 클릭 후 Raw Input Pointer Lock이 가능한 Chrome에서 `RAW INPUT ACTIVE`와 3초 카운트다운이 표시된다.
- [ ] Raw Input을 지원하지 않는 환경에서 표준 Pointer Lock으로 재시도되고 `STANDARD INPUT ACTIVE`가 표시된다.
- [ ] Pointer Lock을 거절하거나 지원하지 않으면 조작 불가 상태로 머물며 페이지 오류가 나지 않는다.
- [ ] `Esc`로 Pointer Lock을 해제했을 때 시간이 멈추고 `PAUSED`가 표시된다.
- [ ] 재개 후 새 3초 카운트다운을 거쳐 남은 시간으로 계속 진행한다.
- [ ] 브라우저 탭 포커스를 잃었다가 돌아와도 세션이 일시정지 상태를 유지한다.
- [ ] 창 크기를 최소 3회 바꿔도 캔버스가 빈 화면/왜곡 없이 다시 그려진다.
- [ ] 한 훈련을 시작·종료·다시 시작하는 흐름을 10회 반복해 콘솔 오류나 누적 캔버스가 없는지 확인한다.
- [ ] 장시간 50 FPS 미만을 유도했을 때 성능 경고가 나타나고, 정상 프레임이 유지되면 사라진다.

## 기록과 복구

- [ ] 완료 결과가 최근 세션과 해당 모드/난이도 개인 최고 기록에 표시된다.
- [ ] 같은 조합을 11회 완료해도 최근 기록은 10개만 남는다.
- [ ] 모드·난이도 필터가 최근 세션과 개인 최고 기록 모두에 적용된다.
- [ ] `기록 삭제` 확인 시 감도/크로스헤어는 유지되고 기록만 삭제된다.
- [ ] `전체 데이터 초기화` 확인 시 감도, 크로스헤어, 기록이 기본값으로 돌아간다.
- [ ] 개발자 도구에서 `valorant-aim-trainer:data`를 손상된 JSON으로 바꾼 뒤 새로고침해도 앱이 기본값으로 복구된다.
- [ ] 로컬 저장소 쓰기를 차단/가득 채운 환경에서 저장 경고가 표시되고 현재 훈련 결과 화면은 유지된다.
